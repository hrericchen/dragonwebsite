# Directives

This folder contains **SOPs (Standard Operating Procedures)** written in Markdown.

Each directive defines:
- **Goal** — What needs to be accomplished
- **Inputs** — What data or context is needed
- **Tools / Scripts** — Which execution scripts to call (from `execution/`)
- **Outputs** — Expected deliverables
- **Edge Cases** — Known pitfalls, API limits, timing notes

## Guidelines
- Write directives like instructions for a mid-level employee.
- Directives are **living documents** — update them when you learn something new (API constraints, better approaches, common errors).
- Don't create or overwrite directives without asking the user first, unless explicitly told to.
